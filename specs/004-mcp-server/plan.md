# Implementation Plan: Embedded MCP Server for atadmin

**Branch**: `004-mcp-server` | **Date**: 2026-05-19 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/004-mcp-server/spec.md`

## Summary

Add `atadmin mcp serve --stdio` — a command that starts an MCP-protocol-compliant stdio server. The server automatically walks the existing Cobra command tree and maps every leaf subcommand into a named MCP tool with typed parameters, enabling any MCP-compatible agent (Claude Desktop, Cursor, Craft Agent) to discover and invoke all atadmin commands natively without reading help text. Tool definitions are derived at startup from flag metadata; tool execution reconstructs CLI arguments and captures output via Cobra's injectable stream mechanism.

## Technical Context

**Language/Version**: Go 1.25  
**Primary Dependencies**:
- `github.com/spf13/cobra` (existing) — source of command tree and flag metadata
- `github.com/mark3labs/mcp-go` — MCP server primitives, stdio transport, JSON-RPC 2.0 handling
- `github.com/spf13/pflag` (existing, transitive) — flag type introspection via `flag.Value.Type()`

**Storage**: N/A — no persistent state; all entities are in-memory for the process lifetime  
**Testing**: `go test ./...` with `httptest.NewServer` for existing API tests; direct buffer/pipe testing for MCP server  
**Target Platform**: macOS and Linux (same as existing atadmin)  
**Project Type**: CLI extension (new subcommand of existing CLI)  
**Performance Goals**: Tool call latency ≤ 2× equivalent direct CLI invocation  
**Constraints**: stdio transport only (no HTTP/SSE in v1); server must survive individual tool call failures  
**Scale/Scope**: Single-user local process; supports 100+ consecutive tool calls without restart

## Key Design Decisions

### D-001: Library — mark3labs/mcp-go

Use `github.com/mark3labs/mcp-go`. It handles JSON-RPC 2.0, stdio transport, initialize handshake, and tool registration. We register tools programmatically at server startup using the library's `server.AddTool()` API.

### D-002: Tool Registration Strategy

At `atadmin mcp serve` startup:
1. Call `NewRootCmd()` to get a fresh Cobra root
2. Walk the command tree via `cmd.Commands()` recursively
3. Exclude: hidden commands, deprecated commands, the `mcp` command subtree, `help` commands
4. For each leaf command: derive tool name, build JSON Schema from flags, register with `mcp-go`
5. For each tool call: create a new `NewRootCmd()` instance, set `SetOut`/`SetErr`, reconstruct `os.Args`-style slice, execute, return buffered output

### D-003: Tool Naming

`strings.Join(pathSegments, "_")` where `pathSegments` are the command names below the root, with `-` replaced by `_` in each segment:
- `atadmin users list` → `users_list`
- `atadmin audit-log list` → `audit_log_list`
- `atadmin api-keys list` → `api_keys_list`

### D-004: JSON Output Auto-Injection

When executing a tool call:
1. Check if underlying command has a `--json` flag: `rootCmd.Find(pathSegments).Flags().Lookup("json") != nil`
2. If yes and the caller did not explicitly set `json: false`: prepend `--json` to reconstructed args
3. Return output with content type `text` (MCP does not have a native JSON content type; the text content contains raw JSON that agents parse)

### D-005: Parameter Reconstruction

For each MCP tool call argument:
- `bool` value `true` → append `--{name}` (flag-only, no value)
- `bool` value `false` → omit
- `int` or `string` → append `--{name}`, `fmt.Sprintf("%v", value)`
- Special case: positional args (e.g., `users_get` needs `<id>`) — detected by checking `cmd.Args` validator and documented as `id` string parameter

### D-006: Diagnostic Logging to File

All MCP server output (startup events, tool call successes, tool call errors) is written to `~/.config/atadmin/mcp.log`. The directory is created with `os.MkdirAll(..., 0o750)` and the log file is opened with `os.O_APPEND|os.O_CREATE|os.O_WRONLY` and mode `0600`. Nothing is written to stdout or stderr after server start — stdout carries only MCP JSON-RPC and any write to stderr would corrupt the protocol stream.

### D-007: Read-Only by Default; Opt-In Mutations

The MCP server exposes only read-only commands by default. Commands whose name matches a mutation verb (`update`, `delete`, `remove`, `bulk`, `add`, `create`) are excluded unless the server is started with `--allow-mutations`. This prevents accidental data modification when AI agents are exploring the tool list. The `--allow-mutations` flag on `atadmin mcp serve` controls this at the server level.

### D-008: Token and Base-URL Excluded from Tool Parameters

The `--token` and `--base-url` persistent flags are excluded from all MCP tool parameter schemas (along with `--help`). This prevents API credentials from appearing in agent request logs or conversation history. Authentication must be pre-configured via `atadmin auth login` before starting the MCP server. If the server's pre-configured credentials are invalid, tool calls return `isError: true` with a hint message directing the user to run `atadmin auth login` and restart the server.
